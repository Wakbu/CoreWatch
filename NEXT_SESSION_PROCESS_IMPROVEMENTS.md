# CoreWatch 다음 작업 추가 항목 — 프로세스 화면

이 문서는 `NEXT_SESSION_TASKS.md`와 함께 다음 작업에서 처리할 필수 항목입니다. 현재는 요구사항만 기록하며 구현하지 않습니다.

## 1. 프로세스별 프로그램 아이콘 표시

현재 프로세스 목록은 글자 중심이라 항목을 빠르게 구분하기 어렵습니다.

수정 대상:

- 실행 파일에서 프로그램 아이콘을 추출해 프로세스 이름 왼쪽에 표시합니다.
- 아이콘을 가져올 수 없는 시스템 프로세스에는 통일된 기본 아이콘을 표시합니다.
- 동일 실행 파일의 아이콘은 매 갱신마다 다시 읽지 않고 경로 기준으로 캐시합니다.
- 아이콘 크기와 이름 사이 간격을 일정하게 유지하고 행의 수직 중앙에 정렬합니다.

완료 조건:

- 일반 앱 아이콘이 Windows 작업 관리자처럼 이름 옆에 식별 가능하게 표시되어야 합니다.
- 아이콘 로딩 때문에 목록 갱신 속도나 스크롤 반응성이 저하되지 않아야 합니다.
- 종료되었거나 접근이 제한된 프로세스에서도 예외 없이 기본 아이콘으로 대체되어야 합니다.

## 2. 프로세스 실시간 갱신 시 화면 깜빡임 제거

현재 갱신 주기마다 목록 전체가 다시 그려지는 것처럼 보여 화면이 미세하게 깜빡입니다.

수정 방향:

- 매 갱신마다 `ItemsSource` 또는 전체 컬렉션을 교체하지 않습니다.
- PID를 키로 기존 항목의 CPU, 메모리, 상태 값만 갱신합니다.
- 새 프로세스만 추가하고 종료된 프로세스만 제거하는 증분 갱신 방식을 사용합니다.
- 백그라운드에서 프로세스 정보를 수집하고 UI 스레드에는 계산이 끝난 변경분만 전달합니다.
- 정렬 순서, 선택 항목, 스크롤 위치가 불필요하게 초기화되지 않도록 유지합니다.
- 아이콘 캐시와 함께 적용해 디스크 접근 및 UI 재생성을 최소화합니다.

완료 조건:

- 실시간 갱신 중 선택 강조, 스크롤 위치 및 현재 정렬이 유지되어야 합니다.
- 목록 전체가 순간적으로 비거나 행 배경이 깜빡이는 현상이 없어야 합니다.
- 실제 실행 화면을 일정 시간 녹화하거나 연속 캡처해 갱신 안정성을 확인합니다.

## 다음 작업 검증 항목

- 일반 앱, 백그라운드 프로세스, 시스템 프로세스의 아이콘 표시를 각각 확인합니다.
- 아이콘이 없는 경우 기본 아이콘으로 안전하게 대체되는지 확인합니다.
- 정렬과 검색을 적용한 상태에서도 실시간 수치가 갱신되는지 확인합니다.
- 갱신 중 선택 항목, 스크롤 위치 및 정렬 방향이 유지되는지 확인합니다.
- 프로세스 화면을 실제로 실행해 시각적 깜빡임이 없는지 검증합니다.
