# 코드 무결성 검증 체크리스트

모든 코드 변경은 아래 과정을 통과해야 완료로 간주한다.

## 1. 변경 범위

- 요구사항과 실제 변경 내용이 일치하는가?
- 사용자 파일이나 관련 없는 코드가 변경되지 않았는가?
- 생성 파일, 임시 파일과 비밀 정보가 저장소에 포함되지 않았는가?

## 2. 코드와 의존성

- 컴파일 오류가 없는가?
- 새 경고가 발생하지 않는가?
- 패키지 버전과 대상 프레임워크가 호환되는가?
- 외부 패키지 라이선스와 알려진 취약점을 확인했는가?
- 취소, 예외, 권한 부족과 미지원 환경을 처리하는가?

## 3. 테스트

- 변경 영역의 단위 테스트 또는 동등한 기능 검증을 수행했는가?
- Debug 빌드가 성공하는가?
- Release 빌드가 성공하는가?
- 프로그램이 시작되고 정상적으로 유지 실행되는가?
- 핵심 화면과 주요 사용자 동작이 정상인가?
- 실패 상황에서도 데이터 손상이나 비정상 종료가 없는가?

## 4. 배포 산출물

- 게시된 실행 파일을 소스 빌드와 별도로 실행했는가?
- 대상 PC에 필요한 런타임과 아키텍처가 명확한가?
- ZIP 내부 파일과 압축 해제 가능 여부를 확인했는가?
- 파일 크기와 SHA-256 해시를 기록했는가?
- 서명되지 않은 실행 파일의 SmartScreen 가능성을 안내했는가?

## 5. 완료 보고

완료 보고에는 수행한 검증, 성공 여부, 미검증 항목 및 알려진 한계를 포함한다. 단순히 "빌드 성공"만으로 전체 기능의 정상 동작을 보장한다고 표현하지 않는다.
