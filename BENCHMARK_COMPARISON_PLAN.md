# CoreWatch 커뮤니티 벤치마크 비교군 설계

## 결론

CoreWatch 점수는 다른 도구의 점수와 직접 합치지 않습니다. 동일한 CoreWatch 벤치마크 버전, 테스트 프로필과 점수 계산식을 사용한 결과만 비교군에 포함합니다.

Geekbench Browser처럼 사용자 제출 결과를 모아 하드웨어별 평균을 만드는 방식은 유효하지만, Geekbench 점수는 Geekbench 자체 기준으로 보정되므로 CoreWatch 점수의 백분위로 변환할 수 없습니다.

## 권장 구조

1. 사용자가 벤치마크를 완료합니다.
2. 전송할 익명 데이터의 미리보기를 표시합니다.
3. 사용자가 명시적으로 동의한 경우에만 결과를 전송합니다.
4. 서버는 동일한 `benchmarkVersion`과 `testProfile`끼리만 집계합니다.
5. 전체, 동일 CPU, 동일 GPU, 비슷한 메모리 구성의 백분위를 반환합니다.
6. 표본이 10개 미만인 그룹은 순위를 표시하지 않고 `표본 부족`으로 표시합니다.

## 전송 데이터

- 스키마와 CoreWatch 벤치마크 버전
- CPU·GPU 모델명과 논리 코어 수
- 메모리 총용량·채널·속도처럼 비교에 필요한 범주 정보
- 운영체제 주 버전과 전원 계획
- CPU·GPU·메모리·디스크 원시 측정값과 계산된 점수
- 최고 온도, 스로틀링 여부와 테스트 완료 여부
- 개인을 식별하지 않는 임의 설치 ID의 서버 측 해시

사용자 이름, Windows 계정, 컴퓨터 이름, IP 주소, 파일 경로, 프로세스 목록, 하드웨어 일련번호는 저장하지 않습니다. 전송은 기본적으로 꺼져 있어야 합니다.

## 서버 검증

- 허용되는 앱·벤치마크 버전 확인
- 원시 측정값과 계산 점수의 재계산
- 물리적으로 불가능한 범위 거부
- 설치 ID와 IP 기반 속도 제한
- 중앙값·MAD 기반 이상치 제외
- 한 설치 ID의 동일 하드웨어·버전 결과는 최신 유효 결과만 대표 표본으로 사용
- 데이터 삭제용 익명 영수증 제공

## API 초안

- `POST /v1/benchmark-results` — 동의한 결과 제출
- `GET /v1/benchmarks/reference?benchmarkVersion=...&cpu=...` — 표본 수, 중앙값, 분위수 조회
- `POST /v1/benchmark-results/delete` — 익명 영수증으로 제출 데이터 삭제

## 구현 순서

1. 앱 내부 제출 스키마, 익명화 미리보기와 로컬 백분위 계산기
2. 자체 호스팅 가능한 ASP.NET Core API와 데이터베이스
3. 동의 UI, 업로드·재시도·오프라인 처리
4. 최소 표본 수, 이상치 제거와 하드웨어 이름 정규화
5. 벤치마크 화면에 전체·동일 하드웨어 백분위와 분포 그래프 표시

## 외부 데이터 정책

- 공개 사이트를 무단 크롤링해 CoreWatch 점수로 변환하지 않습니다.
- 외부 점수는 라이선스와 사용 조건이 허용되는 경우에만 별도의 `외부 참고 점수`로 표시합니다.
- CoreWatch 백분위 계산에는 외부 도구 점수를 사용하지 않습니다.

## 참고

- [Geekbench Browser](https://browser.geekbench.com/)는 전 세계 사용자가 제출한 결과를 공유합니다.
- [Geekbench Processor Benchmark Chart](https://browser.geekbench.com/processor-benchmarks)는 최소 다섯 개의 고유 결과가 있는 프로세서의 평균을 제공합니다.
