# 작업 완료 및 Git 커밋 규칙

- 코드·문서·배포 파일 변경을 완료하면 프로젝트의 필수 무결성 검증을 먼저 수행합니다.
- 검증에 실패한 상태에서는 커밋하거나 완료로 보고하지 않습니다.
- 검증 통과 후 이번 작업에 해당하는 파일만 확인하고 Git에 스테이징합니다.
- 코드·문서·설정·테스트·배포 파일 등 어떤 작업이든 파일 변경이 발생하면 예외 없이 의미가 분명한 메시지로 Git 커밋합니다.
- 커밋하지 말라는 요청이 있는 경우에는 파일 변경 작업 자체를 시작하기 전에 범위를 다시 확인합니다.
- 버전이 변경되거나 기능이 추가·삭제되면 GitHub 메인 페이지의 `README.md` 현재 버전, 주요 기능과 실행·다운로드 안내를 같은 커밋에서 반드시 갱신합니다.
- 완료 보고에는 커밋 해시와 메시지를 포함합니다.
- 설치 없이 실행하는 버전별 publish 보존 규칙은 `PUBLISH_POLICY.md`를 따릅니다.
