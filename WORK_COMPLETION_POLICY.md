# 작업 완료 및 Git 커밋 규칙

- 코드·문서·배포 파일 변경을 완료하면 프로젝트의 필수 무결성 검증을 먼저 수행합니다.
- 검증에 실패한 상태에서는 커밋하거나 완료로 보고하지 않습니다.
- 검증 통과 후 이번 작업에 해당하는 파일만 확인하고 Git에 스테이징합니다.
- 사용자가 별도로 커밋하지 말라고 지시하지 않은 한, 작업 단위가 끝나면 의미가 분명한 메시지로 Git 커밋합니다.
- 완료 보고에는 커밋 해시와 메시지를 포함합니다.
- 설치 없이 실행하는 버전별 publish 보존 규칙은 `PUBLISH_POLICY.md`를 따릅니다.
