# CoreWatch 시스템 최적화 기능 로드맵

## 구현 완료

- ✅ 시작 프로그램 분석, 저장 공간 미리보기, 사용자 승인형 정리와 최적화 전후 비교 (5.3.0)
- ✅ 전원 설정 분석·추천·백업·적용·복원 (5.4.0)
- ✅ 장시간 백그라운드 부하 탐지 (5.5.0)
- ✅ 시작 프로그램 활성화·비활성화 보호 정책과 되돌리기 (5.6.0)
- ✅ 물리·커밋·캐시·커널 풀·페이지 파일·압축 메모리 분석 (5.7.0)
- ✅ TRIM·여유 공간·SMART·파일 시스템 오류 저장장치 진단 (5.10.0)
- ✅ 진단·텔레메트리·상태·벤치마크 통합 JSON/HTML/PDF 보고서 (6.0.0)
- ✅ GitHub Release 기반 자동 업데이트와 SHA-256 검증 (6.1.0)
- ✅ Windows 저장소 센스·예약 저장소 진단과 설정 연결 (6.1.0)
- ✅ 네트워크 지연·패킷 손실·어댑터 오류 진단 (6.1.0)
- ✅ PC 구성과 사용 목적을 반영한 개인화 추천 점수 (6.1.0)
- ✅ 내부 표가 바깥 페이지의 휠 스크롤을 가로채지 않도록 중첩 스크롤 조정 (6.1.0)
- ✅ 네트워크 표 고정 정렬·리사이즈 차단, 상단 조건부 업데이트 배지, 개인화 세그먼트 UI, 하드웨어 바로가기 수정 (6.1.1)

현재 계획된 기능은 모두 완료되었습니다.

## 다음 유지보수 후보

- 다양한 Windows 화면 배율에서 상단 업데이트 배지와 네트워크 표 밀도 점검
- 실제 네트워크 환경의 지연·손실 판정 기준 보정
- 사용자 피드백 기반 UI 밀도와 접근성 개선

## 안전 원칙

- 모든 변경은 미리보기와 사용자 확인 후 실행합니다.
- 관리자 권한은 실제 시스템 변경 시점에만 요청합니다.
- Windows 핵심 서비스, 보안 기능과 업데이트를 임의로 비활성화하지 않습니다.
- 업데이트 확인은 공개 GitHub Release 메타데이터만 조회하고 진단 데이터를 전송하지 않습니다.
- 외부 지연 진단은 사용자가 명시적으로 요청한 경우에만 동작합니다.
- 다운로드한 실행 파일은 신뢰된 릴리스의 SHA-256과 일치하기 전에는 실행하지 않습니다.
