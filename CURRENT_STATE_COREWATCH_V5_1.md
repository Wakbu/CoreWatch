# CoreWatch 5.1 구현 상태

## 활성 프로젝트

- 앱: `src/SystemChecker.App/CoreWatch.V5.1.Release.csproj`
- 자동 검증: `tools/CoreWatch.V5.Verification/CoreWatch.V5.Verification.Final.csproj`
- 대상: .NET 9, WPF, Windows x64

## 이번 변경

- 왼쪽 탐색을 `OVERVIEW → PROCESSES → HARDWARE → BENCHMARK → REPORT`로 재배치했습니다.
- 프로세스 화면을 Windows 11 작업 관리자형 표와 도구 모음으로 다시 구성했습니다.
- 프로세스 CPU/메모리 값을 백그라운드에서 1초마다 다시 수집하며 갱신 시각을 표시합니다.
- CPU/메모리 셀에 절제된 사용량 음영을 적용하고 클릭 선택은 중립 회색으로 변경했습니다.
- 프로세스명/PID/분류/경로 검색, 열 정렬, 보호 분류, 안전 종료를 유지했습니다.
- 하드웨어 화면을 왼쪽 장치 분류와 오른쪽 상세 표의 2단 구성으로 변경했습니다.
- 제목, 표, 셀에 작업 관리자 수준의 여백과 행 높이를 적용했습니다.

## 프로세스 보호 정책

- `보호된 시스템`: Windows 핵심 프로세스와 PID 0~4, 종료 불가
- `Windows 구성요소`: Windows 폴더에서 실행, 종료 불가
- `확인 필요`: 실행 경로 확인 불가, 종료 불가
- `일반 프로그램`: 사용자 확인 후 종료 가능

이 분류는 안전장치이며 악성 코드 판정 기능은 아닙니다.

## 검증 결과

- Release 빌드: 경고 0, 오류 0
- 자동 무결성 검증: 13/13 통과
- 실행 UI 자동화: 탭 순서 통과
- 실행 UI 자동화: 프로세스 갱신 시각이 1초 주기로 변경됨을 확인
- 실행 UI 자동화: 프로세스 CPU/메모리 열 및 하드웨어 분류/상세 2단 화면 확인
