# CoreWatch 다음 작업 필수 항목

현재는 문제와 완료 조건만 기록하며 구현하지 않습니다.

## 1. 왼쪽 메뉴 번호 재지정

프로세스 탭을 두 번째 위치로 이동하면서 기존 번호 `05`가 그대로 남아 순서와 번호가 불일치합니다.

정확한 번호:

1. `01 OVERVIEW`
2. `02 PROCESSES`
3. `03 HARDWARE`
4. `04 BENCHMARK`
5. `05 REPORT`

완료 조건:

- 화면 위치와 표시 번호가 완전히 일치해야 합니다.
- 탭 추가·재배치 시 번호를 문자열로 하드코딩하지 않고 탐색 순서에서 다시 계산하도록 개선합니다.

## 2. 프로세스 표 텍스트 수직 중앙 정렬

첨부 화면에서 `상태`와 `PID`처럼 `DataGridTextColumn`으로 만든 셀의 글자가 행 상단에 붙어 있습니다. CPU와 메모리처럼 별도 템플릿을 사용한 셀과 정렬이 다릅니다.

다음 항목을 모두 행의 실제 수직 중앙에 맞춥니다.

- 프로세스 상태
- PID
- 이름 영역의 아이콘·이름·유형 묶음
- CPU와 메모리 값
- 열 헤더

완료 조건:

- `DataGridCell.VerticalContentAlignment` 설정만 확인하지 말고 실제 `TextBlock` 또는 `ContentPresenter`의 중심을 확인합니다.
- `DataGridTextColumn.ElementStyle` 또는 공통 `CellTemplate`에 `VerticalAlignment=Center`를 명시합니다.
- 실행 화면 캡처로 행 상단·하단 여백이 시각적으로 같은지 확인합니다.

## 3. 하드웨어 표 텍스트 수직 중앙 정렬

첨부 화면에서 장치와 세부 정보 글자가 각 행의 위쪽에 붙어 있고 아래쪽에 큰 빈 공간이 남습니다.

수정 대상:

- 장치 열
- 세부 정보 열
- 하드웨어 표의 모든 `DataGridTextColumn`
- 열 헤더

완료 조건:

- 장치와 세부 정보 텍스트 기준선이 같은 높이에 있어야 합니다.
- 각 행의 글자 위·아래 여백이 육안으로 동일해야 합니다.
- 프로세스 표와 같은 셀 템플릿 및 정렬 규칙을 공유하도록 정리합니다.

## 다음 작업 검수 방법

- 코드 속성만 보고 완료 처리하지 않습니다.
- 프로세스와 하드웨어 화면을 실제 실행하고 각각 캡처합니다.
- 첨부된 문제 화면과 나란히 비교합니다.
- 메뉴 번호 01~05를 UI 자동화로 읽어 순서를 검증합니다.
- 표 행의 경계와 텍스트 위치를 캡처 기준으로 확인한 뒤 완료 처리합니다.

## 참고 이미지

- 첫 번째 이미지: 프로세스가 `05`, 리포트가 `04`로 표시되는 번호 오류
- 두 번째 이미지: 프로세스 상태와 PID가 행 상단에 붙는 문제
- 세 번째 이미지: 하드웨어 장치와 세부 정보가 행 상단에 붙는 문제
