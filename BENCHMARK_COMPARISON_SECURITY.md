# 벤치마크 외부 비교 보안 및 오프라인 정책

## 기본 동작

- 외부 비교는 기본적으로 꺼져 있으며 백그라운드 통신을 하지 않습니다.
- 사용자가 실행 중 체크박스를 켜고 `외부 기준 조회`를 누른 경우에만 한 번 조회합니다.
- 서버 주소와 동의 상태를 디스크나 레지스트리에 저장하지 않습니다.
- 인터넷이 없거나 외부 비교를 끄거나 조회가 실패하면 동일 벤치마크 버전·프로필의 로컬 기록만 비교합니다.

## 외부 통신 제한

- 계정 정보가 없는 HTTPS 절대 주소만 허용합니다.
- GET 요청만 사용하고 벤치마크 결과 업로드 기능은 포함하지 않습니다.
- HTTP, 자동 리디렉션, 쿠키, Windows 기본 인증을 사용하지 않습니다.
- 연결 제한 시간은 6초이며 응답은 최대 1 MiB까지만 읽습니다.
- 요청 쿼리에는 벤치마크 버전·프로필, CPU·GPU 모델, 메모리 용량 구간과 현재 종합 점수만 포함합니다.
- 사용자명, Windows 계정, 컴퓨터 이름, 파일 경로, 프로세스, 하드웨어 일련번호와 설치 ID는 포함하지 않습니다.
- IP 주소를 요청 데이터에 넣지는 않지만 모든 인터넷 연결과 마찬가지로 원격 서버는 연결 과정에서 접속 IP를 확인할 수 있습니다.

## 로컬 데이터

- 로컬 비교 기록은 `%LOCALAPPDATA%\SystemChecker\benchmark-comparison.db`에 저장합니다.
- 현재 벤치마크 버전·테스트 프로필과 호환되는 자신의 기록만 읽습니다.
- 외부 조회 실패는 로컬 기록을 삭제하거나 외부 전송 동의를 자동으로 켜지 않습니다.
